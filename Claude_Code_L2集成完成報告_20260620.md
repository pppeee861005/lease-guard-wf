## 🔑 核心內容詳解

### Prompt 模板 1：基礎掃描

```markdown
【用途】單份租約的標準掃描
【場景】最常見的使用場景
【耗時】15-30 分鐘

【執行步驟】
1. 班組 1：提取基本資訊（JSON）
2. 班組 2：應記載項檢查（23項，表格）
3. 班組 3：不得記載掃描（10項，表格）
4. 班組 4：交叉驗證（5-10項詳解）
5. 班組 5：彙總報告（Markdown）

【質量標準】
- 準確度 ≥98%
- 執行時間 ≤30分
- 規則覆蓋 100%（33項全檢）
```

### Prompt 模板 2：深度分析

```markdown
【用途】對高風險項的集中分析
【場景】已知某份租約有問題，需深度驗證
【耗時】20-40 分鐘

【重點項分析】
每項包含 5 個部分：
1. 條款原文
2. 違規分析 + 法律依據
3. 實際損害估算
4. 法律判斷
5. 建議方案

【輸出】深度分析報告（Markdown）
```

### Prompt 模板 3：批量掃描

```markdown
【用途】大批量租約的並行掃描
【場景】W3 評測期，25份租約同時掃描
【耗時】60-90 分鐘（預期 4-5x 加速）

【執行流程】
1. 讀取所有租約檔案
2. 為每份啟動獨立 L2 掃描
3. 5 班組並行執行
4. 彙總生成：
   - 25 份個別報告
   - 1 份批量統計
   - 1 份性能指標

【加速驗證】
順序執行：25 × 15分 = 375分
並行執行：預期 60-90分
加速倍數：4-5x ✅
```

## 🔄 架構設計

### Claude Code 中的 L2 Agent Teams

```
Claude Code 框架
    ↓
協調器（Lead Agent）
    ├─ 班組 1：結構化層（Agent）
    ├─ 班組 2：應記載層（Task）
    ├─ 班組 3：不得記載層（Task）
    ├─ 班組 4：互查層（Agent，需等待 1-3）
    └─ 班組 5：報告層（Write Tool）
    ↓
結果彙聚
    ↓
最終報告（Markdown）
```

### 組件說明

| 組件 | 實現方式 | 職責 |
|------|--------|------|
| 協調器 | Claude Code Agent | 分配任務、等待結果、彙聚產出 |
| 班組 1-3 | Task / Agent | 並行檢測 |
| 班組 4 | Agent | 交叉驗證（需等待 1-3 完成） |
| 班組 5 | Write Tool | 生成最終報告檔案 |

## 🚨 常見問題排查

### Q1：班組 4 無法重現班組 2、3 的結果

**症狀**：班組 4 說「無此違規」，但班組 2、3 都標記了

**解決方案**：
```markdown
班組 4，請基於以下班組 2、3 的結果進行交叉驗證：

班組 2 結果：Item 9 標記為 🔴 不符
班組 3 結果：Item 25 標記為 🔴 嚴重違規

請重新檢閱這兩項的原始條款並驗證判斷
```

### Q2：報告檔案格式不統一

**症狀**：某些報告缺少快速摘要表或違規詳解

**解決方案**：
```markdown
班組 5，所有報告必須遵循以下格式：

1. Frontmatter（掃描元資訊）
2. 快速摘要表（3項關鍵指標）
3. 班組結果摘要
4. 高風險項詳解（至少5項）
5. 建議方案（Plan A/B/C）

如有缺漏，請補充
```

### Q3：批量掃描某份租約失敗

**症狀**：25份中只有24份成功

**解決方案**：
```bash
# 檢查檔案格式和編碼

# 應輸出 text/plain; charset=utf-8

# 檢查檔案內容
```

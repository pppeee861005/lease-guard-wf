# 🔗 Claude Code 與 L2 集成指南

## 概述

此指南說明如何在 **Claude Code** 環境中整合 **L2 Agent Teams（施工隊協作）** 架構，實現租約的自動化掃描和分析。

---

## 🏗️ 架構設計

### 系統組成

```
Claude Code 框架
    ↓
┌─────────────────────────────────────┐
│  租約哨兵掃描系統（LeaseGuard WF）   │
│                                     │
│  ┌──────────────────────────────┐  │
│  │ L2 Agent Teams 協調器        │  │
│  │ (Lead: 班組 5，Peers: 1-4)  │  │
│  └──────────────────────────────┘  │
│           ↓ 並行分發              │
│  ┌────┬────┬────┬────┬────┐       │
│  │ A1 │ A2 │ A3 │ A4 │ A5 │       │
│  │結構│應記│不得│互查│報告│       │
│  └────┴────┴────┴────┴────┘       │
│           ↓ 彙聚結果              │
│  ┌──────────────────────────────┐  │
│  │ 掃描報告生成器                │  │
│  │ (Markdown + JSON)            │  │
│  └──────────────────────────────┘  │
└─────────────────────────────────────┘
```

### 組件說明

| 組件 | 職責 | 實現方式 |
|------|------|--------|
| **協調器** | 任務分配和結果彙聚 | Claude Code Agent |
| **班組 1-3** | 並行檢測 | 3 個獨立 Claude Code Task |
| **班組 4** | 交叉驗證 | Claude Code Agent（需班組 1-3 輸出） |
| **班組 5** | 報告生成 | Claude Code Write Tool |

## 🚀 快速開始（5 分鐘）

### 方式 1：單份租約掃描（最簡單）

#### Prompt

```markdown
我要使用 L2 Agent Teams 掃描租約。

租約檔案：租約庫/rental_contract_sample.md
規則庫：rules/rules_full.json

請執行以下步驟：

1️⃣ 班組 1（結構化層）：提取基本資訊
   - 出租人、承租人、房屋地址、面積、租期、租金、押金
   - 輸出 JSON 格式

2️⃣ 班組 2（應記載層）：檢查 23 項應記載事項
   - 與 rules_full.json 的應記載項逐一對比
   - 統計符合 ✅ / 部分符合 🟡 / 不符 🔴 的數量
   - 計算符合度百分比

3️⃣ 班組 3（不得記載層）：掃描 10 項禁止條款
   - 檢查是否存在禁止的條款
   - 統計無違規 ✅ / 有違規 🔴 的數量
   - 計算違反度百分比

4️⃣ 班組 4（互查層）：交叉驗證高風險項
   - 對班組 2、3 標記的問題重新檢閱
   - 每項包含：條款原文 → 違規分析 → 法律判斷 → 損失估算
   - 至少深入分析 5 個高風險項

5️⃣ 班組 5（報告層）：生成完整報告
   - 格式：Markdown（檔名：掃描報告_sample_rental_20260620.md）
   - 內容：快速摘要 + 班組結果 + 違規詳解 + 建議方案

質量標準：
- 掃描準確度 ≥98%
- 執行時間 ≤30 分鐘
- 規則覆蓋率 100%（33 項全檢）
```

#### 預期輸出

```
掃描報告_sample_rental_20260620.md
├─ 快速摘要表
│  ├─ 應記載符合度：48%
│  ├─ 不得記載違反度：80%
│  └─ 整體評估：🔴 高度風險
├─ 班組 1：基本資訊（JSON）
├─ 班組 2：應記載項表（23 行）
├─ 班組 3：不得記載項表（10 行）
├─ 班組 4：高風險項詳解（5-10 項）
└─ 班組 5：建議方案（Plan A/B/C）
```

### 方式 2：批量掃描 25 份租約（W3 評測用）

#### Prompt

```markdown
我要使用 L2 Agent Teams 批量掃描 25 份租約。

租約位置：租約庫/ （所有 .md 檔案）
規則庫：rules/rules_full.json

執行步驟：

1. 讀取所有租約檔案（假設已準備 25 份）
2. 為每份租約啟動獨立的 L2 掃描流程
3. 5 個班組並行執行
4. 彙總生成：
   - 25 份個別掃描報告
   - 1 份批量統計報告（含平均符合度、高風險租約數、重複違規統計）
   - 1 份性能指標檔案

質量指標：
- 掃描準確度：≥98%
- 執行時間：記錄每份租約的耗時
- Token 消耗：記錄總消耗
- 加速倍數：計算並行 vs 順序執行的時間對比

期望：4-5x 加速（25份順序需要 25×15=375分，並行預期 60-90分）
```

## 🔧 進階配置

### 場景 1：針對特定風險項的深度分析

如果你已經知道某份租約有特定問題，可以針對性掃描：

```markdown
### 深度掃描高風險項

租約已經過初步掃描，發現以下高風險項：
- Item 25（支付限制）
- Item 28-29（稅賦轉嫁）
- Item 30（單方終止）

請班組 4 進行深度交叉驗證：

對每一項執行以下分析：
1. 條款原文（直接引用）
2. 違規特性 + 法律依據
3. 實際損害估算（金額或權利喪失）
4. 法律判斷（違法無效嗎？）
5. 建議方案（簽署風險？）

輸出：深度分析報告（Markdown）
```

### 場景 2：與律師的協作掃描

如果需要與律師合作驗證掃描結果：

```markdown
### 律師協作驗證掃描

租約掃描報告已生成。請進行以下驗證：

1. 檢查班組 2（應記載項）是否有遺漏
2. 檢查班組 3（不得記載項）是否有假警報
3. 驗證班組 4 的法律判斷是否準確
4. 確認實際損害估算是否合理
5. 對建議方案進行專業評估

輸出：律師驗證報告（附簽名或認證）
```

### 場景 3：持續監控和版本管理

對於長期租約，可以跟蹤變化：

```markdown
### 租約版本變更掃描

原始掃描：掃描報告_sample_rental_20260620_v1.md
更新版本：rental_contract_sample_v2.md

請執行以下對比掃描：

1. 逐條對比原始版和更新版的差異
2. 識別新增、刪除、修改的條款
3. 評估新版本的改進程度
4. 檢查是否引入了新的違規項
5. 更新掃描報告，標記變更點

輸出：版本對比報告
```

## 🛠️ 常見問題排查

### 問題 1：班組 4 的交叉驗證無法重現班組 2、3 的結果

**症狀**：班組 4 說「無此違規」，但班組 2、3 都標記了

**排查步驟**：
1. 檢查班組 4 是否讀取了班組 2、3 的完整輸出
2. 驗證條款原文是否被正確引用
3. 確認規則庫的版本是否一致

**解決方案**：
```markdown
班組 4，請基於以下班組 2、3 的結果進行交叉驗證：

班組 2 結果：Item 9 標記為 🔴 不符（修繕責任完全反向）
班組 3 結果：Item 25 標記為 🔴 嚴重違規（支付限制）

請重新檢閱這兩項的原始條款並驗證判斷
```

### 問題 2：報告檔案格式不統一

**症狀**：某些掃描報告缺少快速摘要表或違規詳解

**排查步驟**：
1. 檢查班組 5 是否收到所有班組的完整結果
2. 驗證 Markdown 格式是否正確

**解決方案**：
```markdown
班組 5，所有掃描報告必須遵循以下格式：

1. Frontmatter（掃描元資訊）
2. 快速摘要表（3 項關鍵指標）
3. 班組結果摘要（班組 1-5 的簡述）
4. 高風險項詳解（至少 5 項，含原文+分析+判斷）
5. 建議方案（Plan A/B/C）

如有缺漏，請補充
```

### 問題 3：批量掃描時某份租約掃描失敗

**症狀**：25 份中只有 24 份成功生成報告

**排查步驟**：
1. 檢查失敗租約的檔案格式（是否為有效 Markdown）
2. 驗證檔案是否被正確讀取
3. 查看是否有編碼問題（UTF-8）

**解決方案**：
```bash
# 檢查檔案格式
file 租約庫/*.md
# 應輸出 text/plain; charset=utf-8

# 檢查檔案是否可讀
head -20 租約庫/[失敗檔案].md
```

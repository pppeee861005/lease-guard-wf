## 🎯 驗收核心原則

### 原則1：Original 欄位 - 零容忍
> **鐵律**：original 與官方 PDF 逐字比對，錯一字即打回

**「不得記載」特點**：
- 條款更簡潔（只說禁止什麼，不詳細說明）
- 邏輯反向（「不得...」而非「應該...」）
- 禁止範圍必須明確，不允許任何模糊

### 原則2：Plain 欄位 - 給19歲看得懂
> **標準**：零法律術語 + 清楚說明「禁止」+ 1-3句簡潔口語 + 「AI起草，待驗收」標記

**「不得記載」特點**：
- 從「房東應該...」改為「房東不能...」
- 強調「無效」或「違法」
- 給出房東常見的違法寫法例子

**高品質 Plain 範例**：
```
Item 26（禁止申報扣稅）：
「房東不能在合約禁止你申報租金來抵稅。
 如果房東寫『承租人不得申報租金』，這條完全無效，
 你照樣可以向稅務機關申報。」

品質指標：
✅ 明確「不能做」
✅ 說明後果（無效、可以）
✅ 簡潔（3句）
✅ 零法律術語
```

### 原則3：Violations 欄位 - 禁止性樣態
> **標準**：3-5 個房東的違法寫法樣態，每個包含 pattern（怎樣違反禁止） + example（違法合約條文或案例） + severity_flag

**「不得記載」特點**：
- pattern：「房東會怎樣違反這項禁止」（不是房客的遭遇）
- example：「房東在合約中的違法條文」或「房客遭遇的真實違法案例」
- severity_flag：通常都是「高」（因為違反即違法）

### 原則4：Severity 欄位 - 禁止性判斷
> **標準**：「不得記載」項目原則上都應為「高」（違法性）

**判斷流程**：
```
該項是否明確禁止？
├─ 是（Item 24-26、28-29、31）→ 「高」（明確違法）
├─ 程序性禁止（Item 27、30）→ 「中-高」（違法但易舉證）
├─ 量化禁止（Item 32）→ 「高」（違反上限）
└─ 兜底禁止（Item 33）→ 「高」（涵蓋一切不公平）
```

## 🔴 重點項目驗收指南

### ⭐ Item 26（禁止申報扣稅）- 常見陷阱項

**重點檢查**：
- [ ] Original 是否清楚說明「禁止限制申報租金費用扣稅」
- [ ] Plain 是否明確告訴讀者「這是你的權利，房東禁止無效」
- [ ] Violations 是否涵蓋：
  - 合約直接禁止申報
  - 簽切結書放棄申報
  - 威脅「申報就漲租」
- [ ] Severity 應為「高」（違反即無效）

**驗收要點**：
- 此項易被房東濫用（為了逃稅）
- 房客常被騙「簽不申報切結」
- 即使簽了也無效，房客仍可申報

### ⭐ Item 32（違約金超額禁止）- 量化邊界關鍵

**重點檢查**：
- [ ] Original 是否清楚定義「不得超過...合理預估」或「月租金1倍」
- [ ] Plain 是否簡潔說明「最多罰1個月租金」
- [ ] Violations 是否涵蓋：
  - 違約金 = 3-6 個月租金的超額案例
  - 「懲罰性」違約金（如存款的50%）
  - 複雜梯級結構的不合理設計
- [ ] Severity 應為「高」（違反民法247條）

**驗收要點**：
- 這是「量化禁止」，邊界明確（月租金1倍）
- 房東常用複雜計算規避監管
- Example 應包含具體超額計算

### ⭐ Item 33（其他不公平條款）- 兜底禁止最複雜

**重點檢查**：
- [ ] Original 是否涵蓋「消費者保護法第52條」的完整禁止範圍
- [ ] Plain 是否說明「任何不公平的條款都無效」
- [ ] Violations 是否涵蓋常見陷阱（需要廣泛但實務）：
  - 限制出入時間
  - 限制訪客
  - 強制購買房東指定物品
  - 額外返還費用
  - 限制布置自由
  - 「非自願續約」條款
- [ ] Severity 應為「高」（兜底禁止）

**驗收要點**：
- 此項最容易被濫用（房東創意設計各種不公平條款）
- Violations 無法窮舉，但應涵蓋常見的5-8個陷阱
- Example 應基於真實案例或判決書

### 13:15-15:30：逐項驗收（10項）

#### 驗收步驟（以 Item 24 為例）

**步驟1：準備**（1分鐘）
```
1. 打開官方 PDF，找到「Item 24：預留默認戶籍遷入條件」
2. 打開規則庫 JSON，找到 "id": "24"
3. 在驗收清單上標記「進行中」
```

**步驟2：驗收 Original 欄位**（2分鐘）
```
1. 複製官方 PDF 的 Item 24 原文
2. 複製 JSON 中的 original 欄位
3. 逐字比對，檢查：
   ✅ 是否完全相同？
   ✅ 有無多字或少字？
   ✅ 禁止範圍是否明確？
   ✅ 有無省略任何內容？

4. 結果：
   ✅ 相同 → 打勾，繼續
   ❌ 不同 → 記錄到 D4_disputed.md，繼續（不修改）
```

**步驟3：驗收 Plain 欄位**（2分鐘）
```
1. 讀 JSON 中的 plain 欄位
2. 自測檢查：
   ✅ 我看得懂嗎？
   ✅ 有沒有法律術語？
   ✅ 是否清楚說明「禁止」性質？
   ✅ 有沒有房東常見的違法寫法例子？
   ✅ 是否 1-3 句簡潔？
   ✅ 有沒有「AI起草，待驗收」標記？

3. 結果：
   ✅ 符合 → 打勾，繼續
   ⚠️ 有問題 → 記錄到 D4_disputed.md，繼續
```

**步驟4：驗收 Violations 欄位**（2分鐘）
```
1. 檢查 violations 陣列：
   ✅ 樣態數：3-5 個？
   ✅ 每個樣態有 pattern（房東違法寫法）+ example（違法合約條文或案例）+ severity_flag？
   ✅ Pattern 是否清楚說明「怎樣違反禁止」？
   ✅ Example 有具體違法條文或真實案例？（不純假設）
   ✅ 邏輯是否合理？

2. 結果：
   ✅ 符合 → 打勾，繼續
   ⚠️ 有問題 → 記錄到 D4_disputed.md，繼續
```

**步驟5：驗收 Severity 欄位**（1分鐘）
```
1. 檢查判定邏輯：
   ✅ 符合禁止性判斷嗎？（「不得記載」通常都是「高」）
   ✅ 有特殊情況嗎？（例如 Item 27、30 可能是「中」）
   ✅ 與該項性質相符嗎？

2. 結果：
   ✅ 符合 → 打勾，繼續
   ⚠️ 有疑義 → 記錄到 D4_disputed.md，繼續
```

**步驟6：完成標記**（1分鐘）
```
1. 在驗收清單上標記「✅ 完成」
2. 如果沒有問題 → JSON 中移除「AI起草，待驗收」標記
3. Git commit：git commit -m "Verify batch3-item-24-預留默認戶籍遷入"
4. 進入下一項
```

**總耗時**：9 分鐘/項 × 10 項 = 90 分鐘 ≈ 1.5 小時

### 重點項特別說明

#### ⭐ Item 26（禁止申報扣稅）- 常見陷阱項

**預期問題**：
- Plain 是否明確說「這是你的稅法權利，房東禁止無效」？
- Violations 是否涵蓋房東常見的欺騙手段（簽切結、威脅漲租）？

**驗收快速檢查**：
```
Plain 應該能說出：
「房東不能在合約禁止你申報租金扣稅。
 如果房東寫『不得申報』，這條完全無效，你照樣可以申報。」

Violations 應該包含：
「合約直接禁止申報」
「要求簽切結放棄申報」
「威脅『申報就漲租或終止租約』」

Severity 應該是：「高」（違反消費者保護法）
```

#### ⭐ Item 32（違約金超額禁止）- 量化邊界關鍵

**預期問題**：
- Original 是否清楚定義「不得超過...月租金1倍」？
- Plain 是否簡潔說明「最多罰1個月租金」？
- Violations 是否包含超額計算的具體例子？

**驗收快速檢查**：
```
Plain 應該能說出：
「房東的違約金不能超過1個月租金。
 如果房東要求賠償2-6個月租金，超額部分可以拒付。」

Violations 應該包含：
「違約金 = 3個月租金以上」（具體數字）
「按房客存款的『懲罰性』百分比計算」

Severity 應該是：「高」（違反民法247條）
```

#### ⭐ Item 33（其他不公平條款）- 兜底禁止最複雜

**預期問題**：
- Original 是否涵蓋「消費者保護法第52條」的完整禁止範圍？
- Violations 是否涵蓋常見的5-8個陷阱（不能窮舉，但應代表性強）？

**驗收快速檢查**：
```
Plain 應該能說出：
「合約有不公平、霸王條款，房東也不能這樣做。
 法律保護租客不受欺負。」

Violations 應該包含常見陷阱（至少涵蓋3-4個）：
✅ 限制出入時間
✅ 限制訪客
✅ 強制購買房東指定物品
✅ 額外返還費用扣款
✅ 限制房間布置自由

Severity 應該是：「高」（兜底禁止）
```

## ⚠️ 常見問題與快速應對

### Q1：驗收時發現 Original 有錯誤，怎麼辦？

**A**：
1. **不要自己修改**（Original 是零容忍的）
2. 記錄到 `D4_disputed.md`
3. 繼續驗收下一項（驗收流程不停頓）
4. EOD 時統計疑義項數量

### Q2：Plain 欄位不夠「禁止性」，可以改嗎？

**A**：
1. 記錄問題到 `D4_disputed.md`
2. 繼續驗收（不停頓）
3. 如果問題很明顯（例如說「應該」而非「不能」），可以記為「打回」
4. 統計完成後決定修改方案

### Q3：Item 33 的 Violations 太難判斷，怎麼辦？

**A**：
1. Item 33 是「兜底禁止」，無法窮舉
2. 檢查是否涵蓋常見的 5-8 個陷阱
3. 如果陷阱代表性強（涵蓋「限制自由」「強制購買」「額外費用」等主題），就可以通過
4. 如果過度簡陋，記錄為「疑義」

### Q4：驗收時間超進度，怎麼辦？

**A**：
1. 優先完成「重點項」（Item 26、32、33）
2. 其他項目如果卡住，記錄疑義，繼續
3. 保持「驗收不可趕工」原則，寧可延遲也要保質量
4. 不超過 16:00，可順延至 D5

## 一眼看懂報告

### 快速摘要三指標

```
應記載符合度  ≥60% ✅ 及格  | <60% 🔴 不及格
不得記載違反度 ≤20% ✅ 安全  | >20% 🔴 高風險
整體評估      🟢低/🟡中/🔴高
```

### 決策樹

```
符合度 ≥60% AND 違反度 ≤20%
  ├─ 是 → 相對安全（可協商修改）
  └─ 否 → 高風險（考慮拒簽）
```

### 三個關鍵看點

| 優先級 | 看什麼 | 在哪找 |
|--------|--------|--------|
| 🔴 P1 | **班組 3** 的違規項目 | 不得記載檢查層 |
| 🟡 P2 | **班組 4** 的法律判斷 | 質量互查層 |
| 🟢 P3 | **班組 5** 的建議方案 | 報告層 |

## 實際案例解讀

### rental_contract_sample.md 的掃描結果

```
應記載符合度：48% → 🔴 不及格
不得記載違反度：80% → 🔴 嚴重違規
整體評估：🔴 高度風險 — 不建議簽署

核心違規 5 項：
  Item 25 → 支付地點限制（強制轉帳）
  Item 28-29 → 稅賦轉嫁（年損失 ~13,440元）
  Item 30 → 房東單方終止（隨時可被逐出）
  Item 32 → 違約金超額（最高 5 個月 + 30 萬）
  Item 33 → 多項不公平條款（年齡/身障歧視等）
```

## 風險等級簡表

```
🟢 低風險（符合度 ≥75%，違反度 ≤10%）
   → 相對安全，可簽署

🟡 中風險（符合度 60-74%，違反度 11-20%）
   → 需協商修改，修改後可簽

🔴 高風險（符合度 <60% OR 違反度 >20%）
   → 不建議簽署，考慮換房東
```

## 最佳實踐 5 條

1. **一定用班組 4** — 不要只看班組 2/3，必看交叉驗證
2. **重點看損失估算** — 嚴重度標籤只是參考，金額損失才是關鍵
3. **優先處理高風險** — 先解決 Item 25/28-29/30，再談其他
4. **保留掃描報告** — 可用於後續談判或舉報
5. **定期更新規則** — 法規每年有調整，檢查 rules_full.json

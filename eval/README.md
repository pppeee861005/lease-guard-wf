## 標注標準

詳見 `annotation-standard.md`（W1 完成）

每份合約標注包括：
- **合約 ID**：唯一標識符（去個資化）
- **風險項**：[條款號] | [風險類型] | [嚴重度] | [原文] | [規範對應項]
- **漏項**：該有但沒有的條款列表
- **標注人員**：姓名
- **標注日期**：YYYY-MM-DD
- **驗收**：✅ 核對無誤 / ⚠️ 需修正

## 評測流程

### 第 1 輪：初始評測（W3 前半）
```
1. 整理 ≥25 份標注完成的合約
2. 運行評測腳本
3. 自動計算：漏報率、誤報率、引用準確率
4. 生成評測報告（含數字 + 每個漏報的覆盤）
5. 記錄至 eval_report_w3_round1_YYYYMMDD.md
```

### 第 2 輪：修復後評測（W3 後半）
```
1. 根據漏報覆盤逐項修復
2. 重新運行評測腳本
3. 比較兩輪數字（不只報好的那輪）
4. 生成最終評測報告
5. 記錄至 eval_report_w3_round2_YYYYMMDD.md
```

## 評測指標

| 指標 | 定義 | 目標值 | 備註 |
|------|------|--------|------|
| **漏報率** | (系統漏掉的風險 / 人工標注的總風險數) × 100% | <5% | 高嚴重度風險漏報 = 最大失敗模式 |
| **誤報率** | (系統報告但人工驗證不存在的風險 / 系統報告總數) × 100% | <3% | 寧可漏報，不要錯報 |
| **引用準確率** | (正確引用原文和規範的風險數 / 風險總數) × 100% | 100% | 鐵律一：雙引用強制 |
| **OCR 成功率** | (識別無誤的字符數 / 總字符數) × 100% | — | 記錄即可，不設目標 |

## 評測報告樣式

每份評測報告必須包含：

```markdown
# 評測報告 #1 | W3 初始評測

**日期**：2026-06-28
**評測集**：25 份合約
**系統版本**：Pipeline v0.1-alpha

## 摘要
- 漏報率：X%
- 誤報率：Y%
- 引用準確率：Z%

## 詳細數據
[表格：每份合約的單項指標]

## 漏報根因分析
[逐一覆盤每個漏報的原因]

## 修復計劃
[針對每個漏報的修法]

## 下一步
[指向 W3 後半的修復工作]
```

## 本地運行評測

```bash
# 安裝依賴
pip install -r requirements-eval.txt

# 運行評測腳本
python eval_script.py --input eval/samples/ --output reports/eval_result.json

# 生成可讀報告
python eval_script.py --input eval/samples/ --format markdown > reports/eval_report_YYYYMMDD.md
```

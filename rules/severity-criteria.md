# 📊 規則庫嚴重度分級標準（Severity Criteria）

**文件用途**：定義租約哨兵規則庫中「高/中/低」三級的邊界判斷標準
**適用範圍**：所有 23 條應記載 + 10 條不得記載事項
**版本狀態**：1.0 初稿（D2 定稿，D3 試驗，D6 終版）

---

## 🎯 核心邏輯

嚴重度分級的核心問題：**「這條規則被違反，承租人會損失什麼？」**

```
🔴 高：直接金錢損失 / 居住權喪失 / 身體安全威脅
      → 違反此規則會立即傷害承租人核心利益

🟡 中：權益受損但可事後救濟 / 訴訟可恢復
      → 違反此規則會造成麻煩，但法律和程序能補救

🟢 低：程序瑕疵 / 資訊透明度不足 / 格式錯誤
      → 違反此規則主要是不便，損失可控或無直接經濟損失
```

---

## 🔴 嚴重度 1：「高」- 直接金錢損失或居住權喪失

### 定義

**任何一種違反會直接導致**：
- 💰 承租人經濟損失（無法全額追回或困難重重）
- 🏠 失去居住地點或居住權（被迫搬家、無處住）
- 😷 身體安全或健康受威脅（長期）
- ⚖️ 被迫進行法律訴訟才能解決

### 判斷檢查清單

```
□ 違反此規則會導致承租人需要掏錢賠償房東嗎？     → 是 = 高
□ 違反此規則會導致承租人被房東強制驅逐嗎？       → 是 = 高
□ 違反此規則會導致承租人失去押金（無法追回）嗎？  → 是 = 高
□ 違反此規則會長期影響承租人健康或安全嗎？       → 是 = 高
□ 承租人需要通過法律訴訟（而非協商）才能救濟嗎？  → 是 = 高
```

### 具體例子

| 條文 | 為什麼是「高」 |
|------|------------|
| **第6點 - 電費計費** | 房東按「最高級距」超收 → 承租人每月損失$200-500 → 年度$2,400-6,000 → **直接金錢損失，且難以追回** |
| **第4點 - 押金約定** | 房東違法扣押金超過上限 → 承租人無法全額追回 → **直接經濟損失** |
| **第8點 - 修繕責任** | 房東將修繕責任全轉嫁給承租人 → 承租人需自費修繕 → **直接金錢損失** |
| **第18點 - 承租人終止權** | 房東不允許因健康危害而終止租約 → 承租人無法離開危險環境 → **健康威脅 + 被迫訴訟** |
| **不得1 - 廣告僅供參考** | 房東用「僅供參考」條款推翻所有約定 → 承租人被騙 → **經濟損失+居住權喪失** |

### 低風險的反例（為什麼不是高）

| 例子 | 為什麼不是「高」 |
|------|------------|
| **第1點 - 審閱期缺漏** | 承租人沒有看合約的時間 → 很嚴重 → **但可以事後撤銷合約** → 屬於「中」 |
| **第20點 - 通知方式** | 房東只用電話通知不用書面 → 容易產生爭議 → **但不是直接經濟損失** → 屬於「低」 |

## 🟡 嚴重度 2：「中」- 權益受損但可事後救濟

### 定義

**違反此規則會造成**：
- 🔄 承租人需要事後透過協商、訴訟、調解才能恢復原狀
- ⏰ 承租人需要花費時間、金錢進行救濟（但救濟概率高）
- 📄 涉及法律程序但結果通常對承租人有利

### 判斷檢查清單

```
□ 承租人可以通過協商解決嗎？                      → 可能 = 中
□ 承租人可以通過訴訟取回權益嗎？（勝率>70%）      → 可以 = 中
□ 違反此規則不會立即導致經濟損失，但會延誤解決？  → 是 = 中
□ 法律明確保護承租人，只是房東違反了？           → 是 = 中
```

### 具體例子

| 條文 | 為什麼是「中」 |
|------|------------|
| **第9點 - 室內裝修** | 房東要求承租人復原，但標準不清 → 承租人可協商或訴訟 → **可救濟，但費時** |
| **第11點 - 修繕義務** | 房東不履行修繕義務 → 承租人可申請法院強制執行 → **法律明確保護，可救濟** |
| **第14點 - 任意終止期限** | 房東違反「提前1個月通知」 → 承租人可主張終止費用 → **可救濟但需打官司** |
| **不得3 - 不得申報租賃費用** | 房東禁止承租人申報稅務 → 承租人可自主申報 → **違法但可自救** |

### 高風險的反例（為什麼不是中）

| 例子 | 為什麼不是「中」 |
|------|------------|
| **電費超收** | 雖然理論上可訴訟 → **但承租人舉證困難** + **訴訟成本高** + **房東實務超收普遍** → 屬於「高」 |
| **押金被違法扣除** | 雖然法律明確 → **但房東常用各種理由延遲退款** + **承租人難以完全追回** → 屬於「高」 |

## 🟢 嚴重度 3：「低」- 程序瑕疵或資訊透明度

### 定義

**違反此規則主要導致**：
- 📋 程序不完整（但實質不傷害承租人）
- 📱 資訊不透明（但承租人可自主查證）
- 🔤 格式或措詞不夠清晰（但沒有實質傷害）
- ⏱️ 效率降低（但不影響最終結果）

### 判斷檢查清單

```
□ 違反此規則主要是「不方便」而非「不公平」？      → 是 = 低
□ 承租人沒有直接經濟損失，但需花時間確認？       → 是 = 低
□ 法律明確，承租人容易自主驗證或補救？          → 是 = 低
□ 違反此規則是「缺漏資訊」而非「隱瞞欺騙」？      → 是 = 低
```

### 具體例子

| 條文 | 為什麼是「低」 |
|------|------------|
| **第20點 - 通知方式** | 房東應該用書面 + 電子 → 只用電話 → **承租人容易確認** → **程序瑕疵** |
| **第2點 - 租期記載** | 房東沒有清楚標註租期 → **承租人可查看存根** → **資訊不透明但可自救** |
| **第23點 - 當事人資料** | 房東沒寫聯絡方式 → **承租人通常已經知道** → **格式瑕疵** |
| **第3點 - 租金支付方式** | 合約沒說明遲交多少天算違約 → **但可協商** → **效率損失，非原則損失** |

### 中等風險的反例（為什麼不是低）

| 例子 | 為什麼不是「低」 |
|------|------------|
| **審閱期缺漏** | 看起來只是「程序瑕疵」 → **但後果是承租人被迫簽署不利條款** → 屬於「中」或「高」 |
| **不提供電費單** | 看起來只是「資訊透明問題」 → **實際上會導致超收無法舉證** → 屬於「高」 |

## 📊 三級對照表 - 快速判斷

| 維度 | 🔴 高 | 🟡 中 | 🟢 低 |
|------|-------|-------|-------|
| **經濟損失** | 直接、無法全額追回 | 可追回但耗時 | 無直接損失或易彌補 |
| **居住権影響** | 喪失 / 被迫搬家 | 有爭議但可保留 | 無影響 |
| **救濟難度** | 困難 / 需訴訟 | 可協商或訴訟 | 容易 / 自救可行 |
| **法律地位** | 明顯違法 | 明顯違法但救濟清楚 | 瑕疵或灰色地帶 |
| **承租人舉證** | 困難 / 房東狡辯空間大 | 相對清楚 | 容易 |
| **承租人成本** | 高（訴訟、搬遷、時間） | 中（協商或訴訟） | 低（確認即可） |
| **實務普遍程度** | 普遍（房東常違反） | 中等 | 少見 |

## 🔀 邊界判例 - 灰色地帶處理

### 判例 1：「電費超收」 vs「電費資訊提供不足」

**情景**：房東收電費，但提供的資訊含糊

```
場景A：房東說「每度3.5元」，符合平均電價
      判定：低 ✅（資訊足夠，計費合法）

場景B：房東說「每度3.5元」，但實際台電是2.8元
      判定：高 🔴（直接超收）

場景C：房東說「電費$1,500」，沒說是幾度
      判定：中 🟡（資訊不透明，但承租人可向台電查詢）
```

**規則**：只要涉及「金錢」，資訊不透明就至少「中」。如果導致實際超收，升級為「高」。

---

### 判例 2：「修繕延遲」 vs「拒絕修繕」

**情景**：房東遲遲不修，或推說「不是我的責任」

```
場景A：房東說「下週會修」，但延誤了一周
      判定：低 ✅（程序延誤，結果預期會修繕）

場景B：房東說「損壞是你自己搞的」，拒絕修繕
      → 實際上是房屋結構問題（如漏水）
      判定：高 🔴（責任轉嫁，健康/安全受威脅）

場景C：房東說「我會修，但要扣維修費$500」
      判定：中 🟡（違法轉嫁成本，但可訴訟追回）
```

**規則**：涉及「健康/安全」的修繕問題一律「高」。責任轉嫁升級為「高」。單純延誤是「低」。

---

### 判例 3：「押金爭議」的三級判定

**情景**：房東要求扣押金

```
場景A：房東說「你損壞了牆壁，要修繕費$2,000」
      但原本就有壁癌（非承租人責任）
      判定：高 🔴（房東欺騙，承租人難以舉證損壞是自己造成）

場景B：房東說「你在牆上打了釘子，必須復原」
      房東要求承租人自費或從押金扣除
      判定：中 🟡（合約應該明確什麼算「原狀」，現在有爭議但可協商）

場景C：房東扣押金$500「清潔費」
      但合約沒說清潔費標準
      判定：中 🟡（責任轉嫁，但數額較小，可追回）
```

**規則**：如果房東舉證困難（承租人易被冤枉）→ 高。有爭議但可協商 → 中。數額小易追回 → 低。

## ✅ 分級標準驗收清單

使用此清單驗證每一條的嚴重度分級：

```
□ 這條規則被違反，承租人會失去什麼？
   - 金錢？ → 高
   - 權益？ → 中
   - 便利性？ → 低

□ 承租人需要什麼努力才能救濟？
   - 訴訟 + 困難的舉證 → 高
   - 協商或簡單訴訟 → 中
   - 自主查驗或協商 → 低

□ 房東容易狡辯嗎？
   - 容易（如電費計算方式模糊） → 高
   - 中等（合約字面意思清楚但有漏洞） → 中
   - 困難（法律明確） → 低

□ 違反此規則在實務中普遍嗎？
   - 很普遍（房東常犯） → 升一級
   - 有時發生 → 維持
   - 少見 → 降一級（因為承租人防範容易）
```

```

  ↓
```

## 📌 重要提醒

### 黑名單（絕對不能判為「低」）

以下條文即使看起來程序簡單，也不能是「低」：

```
❌ 不得2-10（禁止性條款）
   → 任何禁止條款被違反都至少是「中」以上
   → 因為房東違反等同欺騙、隱瞞

❌ 第1點（審閱期）、第4點（押金）、第6點（電費）
   → 這三條是最常被違反的，直接導致經濟損失
   → 任何弱化都是「中」以上

❌ 第18點（承租人終止權）
   → 攸關居住權，任何限制都是「高」
```

### 白名單（通常是「低」但要確認）

```
✅ 第20-23點（通知方式、疑義、其他約定、當事人資料）
   → 除非涉及身份詐騙或隱瞞，否則通常「低」

✅ 格式、措詞、細節性規定
   → 確認沒有實質傷害前提下，可為「低」
```

## 🔧 使用說明

### 何時使用此文件

1. **D2**：Agent 起草時，用此標準為每條提出初判
2. **D3**：逐條驗收時，確認嚴重度是否合理
3. **D6**：標注標準起草時，參考嚴重度影響標註優先級
4. **D7**：試標時，確認標註嚴重度邏輯一致

### 如何引用

在 schema.json 的 `severity` 欄位中填寫「高/中/低」，對應此文件的定義。

如有爭議，記錄到 `disputes.md`：
```
條文ID: X
爭議: [描述為什麼難以判斷]
參考: [本文件的哪個判例]
決策: [D3驗收後填寫]
```

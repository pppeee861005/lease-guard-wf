## 🎯 核心設計理念

### 三大並行場景

1. **多租約並行檢查**（批量處理）
   - 適用：5+ 份租約
   - 加速比：2.5-3x
   - Subagent：3 個

2. **多維度並行分析**（深度檢查）
   - 適用：10+ 個風險維度
   - 加速比：2.5x
   - 分組：P0 財務 / P1 權益 / P2 其他

3. **報告組件並行生成**（輸出優化）
   - 適用：3 段式報告
   - 加速比：2x
   - 組件：風險清單 / 缺失條款 / 轉介資源

### 綜合加速效果

| 場景 | 順序耗時 | 並行耗時 | 加速比 |
|------|---------|---------|--------|
| 單份租約 | 41 分 | 23 分 | 1.78x |
| 25 份租約（W3 評測） | 17.1 小時 | 3.5 小時 | **4.9x** ⚡ |
| 50 份租約（大規模） | 34.2 小時 | 6.5 小時 | **5.3x** ⚡ |

## 🚀 快速開始

### 步驟 1：判斷是否並行
```
任務數量 ≥ 3？
  ↓ YES
無依賴關係？
  ↓ YES
單任務耗時 ≥ 30 秒？
  ↓ YES
結果易整合？
  ↓ YES
✅ 可並行
```

### 步驟 2：選擇並行模式

| 場景 | 推薦模式 |
|------|---------|
| 單份租約（< 3 份） | 雙階段並行（Stage 3+4） |
| 小批量（3-10 份） | 批量並行（3 Subagent） |
| 評測（25 份） | 雙層並行（批量 + 維度） |
| 大規模（50+ 份） | 雙層並行 + 動態調整 |

### 步驟 3：設置 Subagent 數量

| 租約數量 | Subagent 數 | 批次大小 |
|---------|------------|---------|
| 1-2 份 | 1（不並行） | - |
| 3-9 份 | 3 | 1-3 份/批 |
| 10-24 份 | 3 | 3-4 份/批 |
| 25+ 份 | 4-5 | 5-6 份/批 |

## ⚠️ 注意事項

### 適合並行 ✅
- 獨立任務（無依賴）
- 異構任務（不同類型）
- 資源隔離（不競爭）
- 結果可整合（簡單合併）

### 不適合並行 ❌
- 強依賴任務（Stage 2 依賴 Stage 1）
- 資源競爭（同時大量 OCR）
- 複雜整合（需要交叉比對）
- 成本過高（只有 2 個任務）

# 並行決策速查表 ⚡

**用途**：開發時快速判斷「這個任務該並行還是順序執行」

---

## 🚦 3 秒決策流程

```
┌─────────────────────────────────┐
│ 任務數量 ≥ 3？                    │
└─────────────────────────────────┘
         │
    ❌ NO → 不並行（開銷不值得）
         │
    ✅ YES
         ↓
┌─────────────────────────────────┐
│ 任務之間有依賴關係？                 │
└─────────────────────────────────┘
         │
    ✅ YES → 不並行（或設計同步點）
         │
    ❌ NO
         ↓
┌─────────────────────────────────┐
│ 單個任務耗時 ≥ 30 秒？              │
└─────────────────────────────────┘
         │
    ❌ NO → 不並行（加速不明顯）
         │
    ✅ YES
         ↓
┌─────────────────────────────────┐
│ 結果容易整合？                      │
└─────────────────────────────────┘
         │
    ❌ NO → 不並行（整合成本高）
         │
    ✅ YES
         ↓
    ⚡ 可並行
```

---

## ✅ 並行綠燈清單

| 特徵 | 例子 |
|------|------|
| 🔹 **獨立性** | 10 份租約分別檢查 |
| 🔹 **異構性** | 電費分析 + 修繕分析同時進行 |
| 🔹 **資源隔離** | 不同 Subagent 獨立 API call |
| 🔹 **結果可整合** | JSON 列表合併 |
| 🔹 **錯誤隔離** | 租約 1 失敗不影響租約 2-10 |

---

## ❌ 並行紅燈清單

| 特徵 | 例子 |
|------|------|
| 🔸 **強依賴性** | Stage 2 必須等待 Stage 1 完成 |
| 🔸 **資源競爭** | 同時下載 10 個大檔案 |
| 🔸 **複雜整合** | 需要交叉比對的分析 |
| 🔸 **順序語意** | 先判斷風險再評估信心 |
| 🔸 **成本過高** | 只有 2 個任務且每個只需 10 秒 |

| 場景 | 是否並行 | 加速比 | Subagent 數 |

## 🔧 資源配置速查

| 租約數量 | Subagent 數 | 批次大小 | 預計耗時 |
|---------|------------|---------|---------|
| 1-2 份 | 1（不並行） | - | 23 分/份 |
| 3-9 份 | 3 | 1-3 份/批 | 10-15 分 |
| 10-24 份 | 3 | 3-4 份/批 | 20-40 分 |
| 25+ 份 | 4-5 | 5-6 份/批 | 45-60 分 |

---

## ⚠️ 降級條件速查

| 條件 | 動作 |
|------|------|
| Token 消耗達 80% | 停止低優先級並行，只並行 P0 任務 |
| 記憶體 > 90% | 減少 Subagent 數（5→3→2→1） |
| 並行結果不一致 | 立即回退到順序執行 + 記錄 Bug |
| API 限流錯誤 | 降低並行度 + 增加請求間隔 |

---

## 💡 一句話總結

**能並行的核心判斷**：任務多、無依賴、耗時長、易整合 → 並行
**不能並行的核心判斷**：依賴強、競爭資源、難整合 → 順序
